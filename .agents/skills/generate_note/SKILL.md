# Generate Note

根据用户给定的主题，检索 wiki 知识库，整合内容后生成一篇结构化的 Obsidian markdown 笔记。

## 触发时机

- 用户说"结合 wiki 知识生成关于 XXX 的笔记"
- 用户说"帮我写一篇关于 XXX 的笔记"
- 用户说"把 wiki 里关于 XXX 的知识整理成笔记"
- 用户说"生成笔记"并附带主题
- 用户要求"写笔记"、"整理笔记"、"输出笔记"

## 执行流程

### 1. 解析用户需求

从用户输入中提取以下信息：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `topic` | 笔记主题/关键字（必填） | 从用户输入中提取 |
| `target_dir` | 生成笔记的目标目录 | 用户指定，否则为当前工作目录 |
| `filename` | 笔记文件名（不含 .md） | 用户指定，否则根据主题生成 |

如果用户没有指定 `target_dir`，询问用户或默认使用当前目录。
如果用户没有指定 `filename`，根据主题用中文或英文生成合理的文件名（如 `Skill笔记` 或 `Skill_Note`）。

### 2. 检索 wiki 知识库

**步骤 1：读取索引**
读取 `wiki/index.md` 了解知识库结构，定位与主题相关的分类和页面。

**步骤 2：定位相关页面**
根据主题关键词，判断可能相关的 wiki 页面：
- 遍历 `wiki/tools/`、`wiki/concepts/`、`wiki/workflows/`、`wiki/sources/` 等目录
- 通过文件名和 index.md 中的描述判断相关性
- 优先读取与主题直接相关的页面

**步骤 3：深度阅读**
使用 Read 工具读取所有相关 wiki 页面的完整内容，提取与主题相关的知识点。

如果 wiki 页面较多（≥ 100），使用 `obsidian search` 或 `qmd query` 定位相关页面后再读取。

### 3. 学习写作风格

**必须读取风格参考文件**：`references/claude-code-style.md`

同时读取 `Claude Code/` 目录下的 1-2 篇笔记作为实例参考（优先读取与主题相近的笔记，若无则读取 `1、Claude Code入门.md` 和 `3、记忆机制.md`）。

### 4. 整合与生成笔记

将 wiki 中检索到的知识按照风格参考文件的要求整合成一篇完整笔记。

**标题编号规则（核心约束，必须严格遵守）**：
- 一级标题：`# 一、xxx` `# 二、xxx` ...（中文数字 + 顿号）
- 二级标题：`## 1、xxx` `## 2、xxx` ...（阿拉伯数字 + 顿号）
- 三级标题：`### （1）xxx` `### （2）xxx` 或 `### 1.1 xxx`
- **绝对禁止**使用 `# 概述` `## 核心概念` 等无编号标题

**笔记结构模板**：

```markdown
# 一、概述

用 2-4 句话概括主题的核心定义、定位和作用。**关键术语加粗**。
如果涉及多个相关概念，用简短列表说明它们的关系。

> 可选：用引用块补充一句核心观点或注意事项。

# 二、核心概念 / 主要分类

## 1、概念A

详细解释，配合代码块或配置示例。

## 2、概念B

...

### （1）细分点

更详细的说明。

### （2）细分点

...

# 三、使用方法 / 操作指南

## 1、方式一

步骤说明，可配合表格整理参数或命令。

## 2、方式二

...

# 四、对比 / 常见问题 / 注意事项

用表格或列表整理对比信息或常见问题。

# 五、补充

其他相关信息、快捷操作、技巧等。

---

> 笔记中已在正文适当位置使用 `[[页面名]]` 双向链接到相关 wiki 页面。
```

**风格要点**：
- 不使用 YAML frontmatter（个人笔记不需要）
- 大量使用表格整理并列信息（命令速查、参数对比、模式对比等）
- 配置和命令用代码块展示，并标注语言类型
- 关键术语用 `**粗体**` 强调
- 重要提示用 `>` 块引用
- 如有相关 wiki 页面，在**正文适当位置**使用 `[[页面名]]` 双向链接（如提到某个概念时直接链接，不必等到文末）
- 语言使用简体中文
- 叙述口吻客观、说明性，像操作手册

### 5. 写入文件

将生成的笔记内容写入用户指定的目标目录和文件名。

写入前确认：
- 目标目录存在（不存在则创建）
- 文件不会覆盖已有内容（若文件已存在，询问用户或追加内容）

写入后向用户报告：
- 文件完整路径
- 笔记涵盖的主要内容
- 引用的 wiki 页面来源

## 边界与约束

- **不要修改 wiki/ 目录下的任何文件** —— 本 skill 只读取 wiki，不写入 wiki
- **不要修改 Claude Code/ 目录下的任何文件** —— 只读取作为风格参考
- 如果 wiki 中没有与主题相关的知识，如实告知用户，并询问是否基于通用知识生成
- 生成的笔记应聚焦用户指定的主题，不要过度扩展无关内容
