# Claude Code/ 笔记风格参考

本文件详细定义 `Claude Code/` 目录下个人学习笔记的写作风格，供 generate_note skill 生成笔记时精确模仿。

## 一、标题层级系统（必须严格遵守）

笔记采用严格的数字编号标题体系，**绝对不能使用普通的文字标题**。

| 层级 | 格式 | 示例 |
|------|------|------|
| 一级 | `# 中文数字、标题名` | `# 一、概述` `# 二、安装` `# 三、核心概念` |
| 二级 | `## 阿拉伯数字、标题名` | `## 1、基本原理` `## 2、接入MiniMax` |
| 三级 | `### （阿拉伯数字）标题名` 或 `### 数字.数字 标题名` | `### （1）项目级定位` `### 2.1 核心定位` |

**关键规则**：
- 中文数字后紧跟**顿号**`、`，不是空格
- 阿拉伯数字后紧跟**顿号**`、`，不是点号
- 一级标题只能使用中文数字：一、二、三、四、五、六、七、八、九、十
- 如果内容超出十章，继续用十一、十二等
- 二级标题从 1 开始连续编号
- 三级标题在二级标题内部从（1）开始编号，或使用 `x.y` 格式

## 二、笔记结构模板

典型的 Claude Code/ 笔记遵循以下结构：

```markdown
# 一、概述

用 2-4 句话概括主题的核心定义、定位和作用。**关键术语加粗**。
如果涉及多个相关概念，用简短列表说明它们的关系。

> 可选：用引用块补充一句核心观点或注意事项。

# 二、核心概念 / 主要分类

## 1、概念A

详细解释，配合代码块或配置示例。

## 2、概念B

...

### （1）细分点

更详细的说明。

### （2）细分点

...

# 三、使用方法 / 操作指南 / 配置方式

## 1、方式一（或步骤一）

步骤说明，可配合表格整理参数或命令。

## 2、方式二

...

# 四、对比 / 常见问题 / 注意事项

用表格或列表整理对比信息或常见问题。

# 五、补充

其他相关信息、快捷操作、技巧等。

---

## 关联笔记

- [[其他笔记名]] — 简短说明关联原因
- [[另一个笔记]] — 简短说明
```

## 三、表格使用规范

Claude Code/ 笔记**大量使用表格**，这是其最显著的风格特征之一。

**表格场景**：
- 命令速查表（命令 + 说明）
- 参数对比表（参数 + 类型 + 说明）
- 模式对比表（模式 + 行为 + 适用场景）
- 层级对照表（层级 + 位置 + 范围 + 优先级）

**表格写法**：
- 表格前有一行简短的引导语
- 表头使用 **粗体**
- 关键单元格使用 **粗体** 强调
- 表格后常跟 `>` 提示框补充说明

示例：
```markdown
Claude Code 的扩展能力从底层到上层可分为四个层次：

| 层次 | 机制 | 触发方式 | 适用场景 | 复杂度 |
|------|------|----------|----------|--------|
| **MCP Prompts** | MCP Server 暴露的 Prompt | `/mcp__<server>__<prompt>` | 调用外部工具能力 | 中等 |
| **Skill** | SKILL.md 定义的可复用能力 | `/skill-name` 或自动加载 | 封装方法论与工作流 | 低 |

> 旧式 `.claude/commands/*.md` 自定义命令已并入 Skill 机制。
```

## 四、代码块规范

**代码块使用场景**：
- 配置文件（json、yaml）
- 命令行操作（bash）
- 文件内容示例（markdown）
- 代码片段（根据语言标注）

**写法**：
- 必须标注语言类型
- 配置文件前常有一行说明文字
- 命令块中包含注释说明

示例：
```markdown
在本地编辑配置文件`~/.claude/settings.json`，增加：

```json
{
  "env": {
    "ANTHROPIC_API_KEY": "Your API Key",
    "ANTHROPIC_BASE_URL": "https://api.example.com"
  }
}
```

注意：需要将 `ANTHROPIC_API_KEY` 的值填充为复制的 API Key。
```

## 五、强调与提示框

**粗体使用**：
- 首次出现的关键术语：`**Skill**` 是 Claude Code 生态中的核心机制
- 表格表头或关键单元格
- 需要特别突出的概念或警告

**提示框（> 块引用）使用场景**：
- 重要注意事项：`> 注意：Subagent 在单个会话内工作。`
- 补充说明：`> 旧式命令已并入 Skill 机制。`
- 警告信息：`> 父级设置优先，子代理无法覆盖。`
- 核心观点总结

## 六、列表规范

**有序列表**：用于步骤说明、流程描述
```markdown
1. 切换到 `Library` 标签页，选择 `Create new agent`
2. 描述子代理角色和触发时机
3. 选择工具权限
```

**无序列表**：用于并列要点、特性罗列
```markdown
- **上下文保护**：将探索过程隔离到独立上下文
- **权限隔离**：通过限制可用工具强制执行约束
- **专业化**：每个子代理针对特定领域深度定制
```

## 七、双向链接

- 使用 `[[页面名]]` 链接到相关笔记
- 链接通常在文末的 `## 关联笔记` 区域集中列出
- 正文中也可以穿插使用，但不要太频繁
- 链接到 wiki 页面时使用 wiki 中的页面名

## 八、截图引用

- 格式：`![[image-xxx.png]]` 或 `![[Pasted image xxx.png]]`
- 截图紧跟在说明文字之后
- 生成笔记时，如果没有实际截图，**不要编造截图引用**
- 如果原 wiki 内容中有截图引用，可以保留

## 九、叙述口吻

- **客观、说明性**：像操作手册或教程，而非闲聊
- **引导性语句**：使用"你可以..."、"适合..."、"当...时"
- **对比性说明**：使用"前者...后者..."、"A 适合...B 适合..."
- **避免口语化**：不使用"啦"、"呢"、"吧"等语气词
- **避免第一人称**：不使用"我认为"、"我觉得"

## 十、其他细节

- **不使用 YAML frontmatter**：个人学习笔记不需要 frontmatter（wiki 页面才需要）
- **分割线 `---`**：用于分隔大章节或正文与关联笔记区域
- **脚注/锚点**：`[[#^anchor-name]]` 用于内部引用，但在生成笔记时通常不需要
- **文件命名**：笔记文件名通常以数字开头，如 `1、Claude Code入门.md`、`2、Slash命令.md`
