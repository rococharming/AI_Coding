# 一、Obsidian 设置汇总

## 1、主题

- 使用 **Things** 主题（深色风格），白色/浅色页面不喜欢

## 2、CSS 代码片段

### （1）body-text-indent.css — 段落首行缩进

**功能**：阅读模式下，所有段落自动首行缩进 2 个汉字宽度。

**说明**：Markdown 中按 Tab 会被解析为代码块，因此在 Obsidian 中不能通过 Tab 实现首行缩进，必须使用 CSS snippet。源文件保持干净，缩进只在渲染时生效。

```css
.markdown-preview-view p {
  text-indent: 2em;
}
```

### （2）inline-code-highlight.css — 行内代码高亮

**功能**：让 `` `包裹的内容` `` 以紫色高亮显示，与普通文字区分开。

```css
.cm-inline-code {
  color: #e0a0ff !important;
  background-color: rgba(180, 100, 255, 0.15) !important;
  padding: 1px 4px;
  border-radius: 3px;
}

.markdown-preview-view code {
  color: #e0a0ff !important;
  background-color: rgba(180, 100, 255, 0.15) !important;
  padding: 1px 4px;
  border-radius: 3px;
}
```

## 3、安装的社区插件

### （1）Advanced Tables

增强 Markdown 表格的编辑体验：

| 功能 | 说明 |
| ---- | ---- |
| 自动格式化 | 自动对齐表格竖线 |
| Tab/Enter 导航 | Tab 右移单元格，Enter 换行 |
| 列对齐 | 设置左/中/右对齐 |
| 排序 | 按指定列排序 |
| 导出 | 导出为 CSV |

**创建表格方法**：输入 `|` + 标题 + **Tab**（跳到下一个单元格） + 标题 + **Tab** ... + **Enter**（开始填写数据行）。

**注意**：该插件**不支持拖拽调整列宽**。如需精确列宽，需使用 HTML 表格写法。

## 4、Markdown 笔记标题编号规范

| 层级 | 格式 | 示例 |
| :--- | :--- | :--- |
| 一级标题（#） | 中文序号 + 顿号 | `一、` `二、` `三、` |
| 二级标题（##） | 阿拉伯数字 + 顿号 | `1、` `2、` `3、` |
| 三级标题（###） | 括号数字 + 顿号 | `（1）、` `（2）、` `（3）、` |

## 5、Tips

- 显示大纲面板：`Cmd + P` → 搜索 "outline" → 选择"大纲：显示大纲"
- 单元格内换行：使用 `<br>` 标签，不能直接按回车
- Markdown 源文件不要手打 Tab 缩进，用 CSS snippet 在渲染时处理
- Things 主题默认三级标题是蓝色 `#2e80f2`，已通过 snippet 改为白色 `#dadada`（与一级、二级标题一致）

## 6、Markdown 代码块嵌套问题

**问题**：当外层代码块内部还包含代码块时，内容会"脱离"出外层代码块，导致渲染异常。

**原因**：Markdown 解析器遇到与开头相同数量的反引号时，就会认为代码块结束了。如果外层用 3 个反引号（```` ``` ````），内层也用了 3 个反引号，解析器会把内层的反引号误认为外层结束标记。

**解决方法**：外层代码块使用更多反引号（如 4 个），与内层区分开：

```markdown
````markdown
外层内容，内部可以安全使用 3 个反引号的代码块：
```bash
echo 这是内层代码块
```
````
```

关键规则：**外层反引号数量 > 内层反引号数量**。

## 7、Obsidian 内部链接（块引用）

**链接到标题**：
```markdown
[[#标题名称]]                  # 当前文件内的标题
[[笔记名#标题名]]              # 其他文件的标题
```

**链接到块（更精确，推荐）**：
```markdown
在目标行末尾添加块 ID：## 某标题 ^my-id
引用：[[#^my-id]]
自定义显示文字：[[#^my-id|显示文字]]
```

**块 ID 规则**：`^` 后接英文/数字/连字符，本笔记内唯一。

Tips：输入 `[[` 后 Obsidian 会自动补全文件名和标题名。
